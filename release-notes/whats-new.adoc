= Was ist neu in AI Data Engine
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Erfahren Sie mehr über die neuesten Funktionen, die in AI Data Engine (AIDE)-Releases für das KI-Datenmanagement verfügbar sind, einschließlich Katalogisierung, Workspace-Management und flexibler Bereitstellungsoptionen.



== Was ist neu in AIDE 1.0.0

AIDE 1.0.0 (veröffentlicht am 30. April 2026) führt AIDE als reine Software-Bereitstellung auf Servern von Drittanbietern ein. Mit dieser Bereitstellungsoption können Sie AIDE-Software auf Ihren eigenen RHEL 9.7-Servern ausführen, ohne dass NetApp Data Compute Nodes (DCNs) erforderlich sind. In dieser Version unterstützt die AIDE-Software auf Servern von Drittanbietern metadatenorientierte Funktionen.

.link:../install-aide-server/aide-mde-server-hardware-requirements.html["Option für die vom Kunden verwaltete Hardwarebereitstellung"]
Die Bereitstellungsoption der AIDE-Software auf Servern von Drittanbietern bietet Ihnen die Flexibilität, Ihre eigene Hardware mit den Metadatenverwaltungsfunktionen von AIDE zu nutzen und gleichzeitig die zugrunde liegende Hardware und das Betriebssystem selbst zu verwalten. Zu den wichtigsten Merkmalen dieses Bereitstellungsmodells gehören:

* Drei-Knoten homogen Cluster erforderlich
* Sie installieren das Betriebssystem und verwalten die Sicherheitspatches des Betriebssystems
* NetApp bietet AIDE-Software-Stack und Support
* Erfordert ONTAP 9.18.1 oder höher
* Erfordert ein ONTAP AFX-Speichersystem


.link:../get-started/learn-about.html#deployment-options["AIDE Metadata Engine grundlegende Unterstützung"]
AIDE-Software auf Servern von Drittanbietern unterstützt die Kernfunktionen der Metadata Engine von AIDE und ermöglicht es Ihnen, Ihre Metadaten für KI-Workloads zu katalogisieren und zu organisieren.

In dieser Version unterstützt die AIDE-Software auf Servern von Drittanbietern keine GPU-abhängigen Dienste wie Vektorisierung, OCR und Anreicherung, und sie unterstützt keine Datensammlungen, Vektoreinbettungen oder RAG-Endpunkte für die semantische Suche. Ein Upgrade-Pfad zu den vollständigen AIDE-Funktionen wird unterstützt, sobald diese verfügbar sind.

.link:../get-started/aide-architecture.html#metadata-engine["Unterstützte Funktionen der Metadata Engine"]
AIDE Software auf Servern von Drittanbietern bietet folgende Metadatenfunktionen:

* Arbeitsbereichserstellung und -verwaltung
* Automatisierte Metadatenextraktion und Katalogisierung
* Metadatensuche und -filterung mithilfe von REST-APIs
* Data Sync für automatisierte Datenaktualität
* Metadatenexportfunktion


.link:../metadata-search/explore-metadata.html["Metadata Engine Suche und Browsen"]
In AIDE 1.0.0 ist die Metadata Engine-Funktionalität in der AI Data Engine Console verfügbar. Sie können Metadaten aus einer zentralen Ansicht über alle Arbeitsbereiche hinweg suchen, durchsuchen, filtern und exportieren. Sie können Metadaten über eine neue Seite in der AIDE Console erkunden, die mit allen AIDE-Softwareinstallationen auf Servern von Drittanbietern enthalten ist. Diese Seite kann verwendet werden, nachdem die Erstellung des Arbeitsbereichs und die Metadatenextraktion abgeschlossen sind.



== Was ist neu in der ersten Version von AIDE 9.18.1U0/U1

AI Data Engine (AIDE) 9.18.1 U0/U1 ist die erste Version der NetApp-Plattform für KI-Datenmanagement. Diese Version führt eine Metadata Engine und Management-Workflows ein, mit denen Unternehmen unstrukturierte Daten für KI-Workloads katalogisieren und organisieren können und die die Grundlage für erweiterte Governance- und Vektorisierungsfunktionen bieten. Erweiterte Governance (Guardrails) und Vektorisierung stehen Kunden mit den entsprechenden AI Data Engine Lizenzen zur Verfügung.

AIDE 9.18.1U0 führt die folgenden grundlegenden Funktionen ein:

.link:../get-started/aide-architecture.html#metadata-engine["Metadata Engine für die KI-Katalogisierung"]
Die erste Version beinhaltet eine Metadata Engine, die Dateien und Objekte über ONTAP Cluster hinweg katalogisiert.

Zu den wichtigsten Merkmalen gehören:

* Automatisierte Extraktion von Metadaten (Kern- und erweiterten Attributen, Objekt-Tags) aus lokalen und entfernten ONTAP Volumes auf Peered-Clustern.
* Zentralisierte Abfrage- und Filter-REST-APIs für Anwendungen, die einen globalen Überblick über Unternehmensdaten benötigen.
* Skalierbarer Metadatenspeicher.
* Automatische Metadatenextraktion, die während der Workspace-Erstellung ausgelöst wird.


.link:../workspaces/create-workspace.html["Arbeitsbereichsverwaltung"]
Arbeitsbereiche ermöglichen die logische Gruppierung von Datenquellen (Volumes) für AI-Projekte.

Die erste Version unterstützt Folgendes:

* Erstellung von Arbeitsbereichen, die sich über lokale und entfernte ONTAP Volumes erstrecken (mithilfe von Cluster Peering).
* Zuordnung von Zugriffskontrollen zu Arbeitsbereichen, Unterstützung von Mehrbenutzer- und mandantenfähigen Umgebungen.
* Automatische Metadatenextraktion und Katalogbefüllung bei der Erstellung des Workspace.


.link:../get-started/aide-architecture.html#data-sync["Data Sync für automatisierte Datenaktualität"]
Data Sync hält Metadatenkataloge und Datensammlungen aktuell, wenn sich Quelldaten ändern, ohne manuelle Eingriffe.

Zu den wichtigsten Merkmalen gehören:

* Automatisierte Synchronisierung von Daten aus entfernten oder lokalen ONTAP Clustern mittels richtliniengesteuerter SnapMirror-Replikation.
* Inkrementelle Aktualisierungen, die nur geänderte Daten weitergeben und den Aufwand reduzieren.
* Konfigurierbare Aktualisierungsintervalle pro Arbeitsbereich.
* Überwachung des Synchronisierungsstatus und der Aktivität auf Workspace-Ebene.


.link:../install-setup/cluster-setup-afx.html["Cluster-Einrichtung und -Verwaltung"]
Die erste Version umfasst die folgenden Arbeitsabläufe:

* Erkennung und Hinzufügung von Data Compute Nodes (DCNs) während der Cluster-Einrichtung.
* Erstellung dedizierter Metadatenspeicher-VMs für die Metadata Engine.
* Konfiguration der Data Engine Service-Schnittstellen für den clusterweiten Metadatenzugriff.
* Peering mit anderen ONTAP Clustern, um die Katalogisierung von Metadaten über den gesamten Datenbestand hinweg zu erweitern.


.link:../install-setup/configure-oidc.html["OpenID Connect (OIDC) Authentifizierung"]
* OIDC/OAuth-basierte Authentifizierung für sicheren Zugriff auf ONTAP System Manager und Data Engine Console mit Microsoft Entra ID und Active Directory Federation Services (ADFS).
* Rollenbasierte Zugriffskontrollen für Arbeitsbereichs- und Metadatenverwaltung.


.link:../vectorization/zero-to-rag.html["Erweiterte Datenkuratierungs- und Governance-Funktionen"]
Die folgenden Funktionen stehen mit dem Update AIDE 9.18.1U1 für Kunden mit den entsprechenden AIDE-Lizenzen zur Verfügung:

* *Vektorisierung und RAG*: Erstellung von Datensammlungen, Einbettungen und Abrufendpunkten in der AI Data Engine Console, unter Verwendung von Metadaten aus AIDE Workspaces.
* *Leitplankenbasierte Governance*: Definition von Leitplankenrichtlinien in der AI Data Engine Console und Zuordnung dieser Richtlinien zu Arbeitsbereichen im ONTAP System Manager.

* AFX 1K Speicherknoten
* NetApp Datenverarbeitungsknoten
