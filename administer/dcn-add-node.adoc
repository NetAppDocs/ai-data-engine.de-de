= Fügen Sie Ihrem AI Data Engine Cluster Datenrechenknoten hinzu
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können Datenrechenknoten im Rahmen der Erstellung eines neuen AI Data Engine (AIDE)-Clusters oder der Erweiterung eines bestehenden Clusters hinzufügen. Der Workflow umfasst das Erkennen und Konfigurieren der Knoten mithilfe von ONTAP System Manager.



== Bereiten Sie das Hinzufügen von Knoten vor

Beachten Sie Folgendes beim Hinzufügen von Datenrechenknoten.

.Minimale Anzahl an Datenrechenknoten
Für einen neuen AIDE Cluster werden genau drei verfügbare Datenrechenknoten benötigt.

.Hardwareinstallation und Adressierbarkeit
Stellen Sie sicher, dass die folgenden Voraussetzungen erfüllt sind:

* Die neue Datenverarbeitungshardware ist in einem Rack montiert, eingeschaltet und mit den Cluster-Switches verkabelt.
* Für die Datenrechenknoten steht ein IP-Adressraumbereich zum ONTAP Backend-Subnetz zur Verfügung.
* Der ONTAP Cluster wurde initialisiert und ist vom Cluster Management LIF aus erreichbar.


.System Manager-Anmeldeinformationen
Sie benötigen _storage administrator_-Berechtigungen, um AIDE-Cluster-Erstellungs- oder Erweiterungsaufgaben durchzuführen.

.Softwarekompatibilität
Prüfen Sie anhand der folgenden Dokumentation, ob die Hardware- und Softwareversionen Ihrer Datenverarbeitungsgeräte mit Ihrem ONTAP Cluster kompatibel sind:

* link:../administer/upgrades-aide-summary.html["AIDE-Software ist mit der ONTAP Version kompatibel"].


Während des Hinzufügens von Knoten überprüft der System Manager, ob auf den neuen Knoten eine Softwareversion ausgeführt wird, die kompatibel ist mit:

* Die effektive Version des ONTAP-Clusters, wenn dies der erste hinzugefügte Datenrechenknoten ist.
* Die Version des bestehenden Datenrechenclusters, falls bereits Datenrechenknoten vorhanden sind.


Wenn ein Knoten inkompatibel ist:

* Im Dialogfeld *Hinzufügen* wird neben dem betroffenen Datenberechnungsknoten ein Fehler angezeigt.
* Sie müssen zuerst die Software des Datenrechenknotens (oder ONTAP, je nach Bedarf) auf eine kompatible Version aktualisieren.




== Fügen Sie die Datenberechnungsknoten hinzu

Fügen Sie Datenrechenknoten im Rahmen der Erstellung eines neuen AIDE Clusters oder der Erweiterung eines bestehenden Clusters hinzu.

.Schritte
. Wählen Sie im System Manager den *Dashboard* im Navigationsbereich und dann die Karte *Health* aus.
. Bestätigen Sie, dass Knoten zum Hinzufügen vorhanden sind, und wählen Sie *Details anzeigen*, um die Liste anzuzeigen.
+
Die Liste enthält gefundene Knoten, die noch nicht Teil des AIDE-Clusters sind.

. Alternativ können Sie *Cluster* und *Overview* sowie die Registerkarte *Data compute* auswählen, um die Liste anzuzeigen.
. Wählen Sie am unteren Rand der Seite „Datenberechnung“ oberhalb der Knotenliste *Hinzufügen* aus.
. Im Dialogfeld *Knoten hinzufügen* wählen Sie die Data Compute Nodes aus, die Sie hinzufügen möchten.
. Wenn Sie zum ersten Mal Knoten hinzufügen und kein Backend-Subnetz existiert, wählen Sie *Subnetz hinzufügen* und geben Sie Folgendes an:
+
** Subnetzname (für interne Verwendung)
** Subnetzadresse und Maske
** IP-Adressbereich für Datenrechenknoten und ONTAP-Knoten im Backend-Netzwerk
+
Der System Manager prüft, ob der Adressbereich genügend freie IP-Adressen für alle hinzuzufügenden Datenrechenknoten, alle ONTAP-Knoten im Cluster sowie zusätzliche Floating IPs auf Clusterebene enthält, die für die Kommunikation zwischen Datenrechenknoten und ONTAP verwendet werden.



. Führen Sie die folgenden Schritte unabhängig davon aus, ob Sie das Backend-Subnetz hinzugefügt haben oder es bereits existiert:
+
.. Überprüfen Sie die verfügbaren IP-Adressen.
.. Wählen Sie bei Bedarf *Subnetz bearbeiten* aus und erweitern Sie den IP-Bereich.
+
*** Der Bereich kann nur erweitert werden. Das Verkleinern oder Ändern des Subnetzes wird nicht unterstützt.
*** Eine Änderung des Subnetzes oder des IP-Bereichs kann die Neuerstellung des zugrunde liegenden Kubernetes-Clusters auf den Datenrechenknoten erforderlich machen und mehrere Minuten dauern.




. Optional können Sie die Data Engine-Service-Schnittstelle konfigurieren, indem Sie Folgendes angeben:
+
** Dienst-IP-Adresse
** Netzwerkmaske
** Gateway (falls für Ihre Umgebung erforderlich)
+
Die einzelne IP-Adresse wird über die Datenrechenknoten verteilt und als Frontend-Adresse für AIDE Console und zugehörige APIs verwendet.



. Überprüfen Sie die ausgewählten Knoten, das Backend-Subnetz und die Einstellungen der Data Engine-Service-Schnittstelle.
. Wählen Sie *Hinzufügen* und warten Sie, bis der Vorgang abgeschlossen ist. System Manager führt die folgenden Aktionen aus:
+
** Fügen Sie die ausgewählten Knoten dem Datenrechencluster hinzu
** Stellen Sie das Backend-Netzwerk bereit und fügen Sie Knoten dem Kubernetes-basierten Data Compute Cluster hinzu
** Interne Metadaten für die Datenrechenknotenerkennung aktualisieren


. Nach Abschluss wählen Sie *Cluster* und *Overview* aus und bestätigen Sie:
+
** Unter *Datenberechnung* werden die neuen Datenberechnungsknoten als Teil des Clusters online angezeigt
** Alle Knoten sind *Healthy*
** AIDE Version wird für jeden Knoten korrekt angezeigt
** Überprüfen Sie, ob die *Health*-Karte im Dashboard die aktualisierte Knotenanzahl anzeigt.


. (Optional) Nachdem alle Knoten dem Cluster beigetreten sind, überprüfen Sie, ob alle Dienste ordnungsgemäß starten.
+
.. Überprüfen Sie den Kubernetes-Pod-Status:
+
[source, bash]
----
kubectl get pods -n aide-system
----
.. Prüfen Sie, ob alle Pods den Status *Running* oder *Completed* aufweisen.
