= AI Data Engine einrichten und konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Als ONTAP-Speicheradministrator nutzen Sie ONTAP System Manager, um AI Data Engine (AIDE) in Ihre AFX-Umgebung zu integrieren. Nachdem Sie die Installation Ihrer Datenrechenknoten abgeschlossen haben, kann ONTAP diese erkennen und zu einem Cluster hinzufügen. Sie können AIDE-Knoten während der anfänglichen AFX-Cluster-Einrichtung hinzufügen oder sie zu einem bestehenden AFX-Cluster hinzufügen und anschließend die erforderliche AIDE-Konfiguration abschließen.

.Bevor Sie beginnen
* Sie haben NetApp DCNs oder Server von Drittanbietern installiert und mit den Cluster-Switches verbunden. Siehe Installationsdokumentation für link:../install-aide-server/install-workflow.html["AIDE software auf Servern von Drittanbietern"] oder link:../install-dcn/install-workflow.html["NetApp DCNs"] für weitere Informationen.
* (Für Server von Drittanbietern) Sie haben link:../install-setup/install-aide-as-software.html["AIDE-Softwareinstallation"] auf jedem Server abgeschlossen. ONTAP kann diese Server dann als Rechenknoten erkennen und Sie können sie einem Cluster hinzufügen.




== Richten Sie AIDE ein und integrieren Sie es in ein AFX-Speichersystem

Es gibt zwei Möglichkeiten, AIDE einzurichten und in ein AFX-Speichersystem zu integrieren. Wählen Sie die Option, die für Ihre Umgebung geeignet ist.



=== AIDE mit einem neuen AFX-Cluster einrichten

Wenn Sie AIDE im Rahmen einer erstmaligen AFX-Systembereitstellung einrichten, ist der AIDE-Einrichtungsprozess Teil des Standard-Workflows für die AFX-Cluster-Einrichtung. Während der ONTAP-Cluster-Einrichtung erkennt der System Manager automatisch die angeschlossenen Rechenknoten und integriert sie in die Clusterkonfiguration.


NOTE: Sie müssen link:../install-setup/configure-oidc.html["OpenID Connect-Authentifizierung konfigurieren"] aktivieren, um den Zugriff auf AIDE Console zu ermöglichen.



=== AIDE mit einem vorhandenen AFX-Cluster einrichten

Wenn Sie Rechenknoten (entweder NetApp DCNs oder Server von Drittanbietern) zu einem bestehenden AFX-Speichersystem hinzufügen, müssen Sie die Knoten hinzufügen und die Einrichtung Ihres AIDE Clusters abschließen.



==== Fügen Sie dem Cluster Rechenknoten hinzu.

ONTAP erkennt die mit Ihrem Clusternetzwerk verbundenen Rechenknoten und zeigt sie im System Manager an. link:../administer/dcn-add-node.html["Fügen Sie Ihrem Speichercluster Rechenknoten hinzu, um ihn in einen AIDE Cluster zu verwandeln."].



==== Schließen Sie die Einrichtung Ihres AIDE Clusters ab

Nach dem Hinzufügen der Rechenknoten und dem Abschluss der ersten Netzwerkkonfiguration können Sie die Einrichtung Ihres neuen AIDE Clusters abschließen.

.Über diese Aufgabe
Auf der Seite wird ein Willkommensfenster mit den notwendigen Konfigurationsschritten angezeigt. Links neben jedem bereits abgeschlossenen Schritt bzw. jeder abgeschlossenen Aktion befindet sich ein Häkchen. Ein Pfeil (image:icon_arrow.gif["blauer Pfeil, der nach rechts zeigt"]) kennzeichnet noch nicht abgeschlossene Aktionen. Für jede der noch nicht abgeschlossenen Aktionen gilt Folgendes:

* Wählen Sie den aktivierten Titellink aus und schließen Sie die Einrichtungsaktion für Ihre Umgebung ab.
* Wenn der Link deaktiviert ist, bewegen Sie den Mauszeiger über den Titel, um die erforderliche Aktion zum Aktivieren anzuzeigen und mit der Einrichtung fortzufahren.


Jeder Schritt enthält, sofern möglich, einen Link zu weiterer Dokumentation. Einige Konfigurationen sind möglicherweise bereits in der AFX-Cluster-Einrichtung in https://docs.netapp.com/us-en/ontap-afx/["AFX-Dokumentation"^] enthalten.

.Schritte
. Wählen Sie *Link-Aggregation-Gruppen und VLANs konfigurieren*.
+
Falls für Ihre Umgebung erforderlich, können Sie LAGs und VLANs auf Clusterebene einrichten. Durch Auswahl dieser Option wird die Ethernet-Ports-Seite geöffnet.

. Wählen Sie *Netzwerkprotokolle konfigurieren* aus.
+
Sie können die Storage VM (SVM) Netzwerkprotokolle konfigurieren.

. Wählen Sie *Update data compute software* aus.
+
Wenn ein AIDE software-Image zur Aktualisierung verfügbar ist, wird diese Option aktiviert. Wählen Sie sie aus, um die Seite für die Softwareaktualisierung zu öffnen. Weitere Informationen finden Sie unter link:../administer/update-aide-firmware-software.html["AIDE-Software aktualisieren"].

. Wählen Sie *Configure data engine networking*.
+
Diese Option zeigt die Seite zur Konfiguration der AI Data Engine-Netzwerkadresse an.

. Wählen Sie *Intercluster-Netzwerkschnittstellen erstellen*
+
Diese Option öffnet die Seite zur Konfiguration der Intercluster-Netzwerkschnittstellen als Teil eines zweistufigen Verfahrens, wenn Sie beabsichtigen, Remote-Daten zu katalogisieren.

. Wählen Sie *Peer mit anderen ONTAP storage systems*.
+
Nach der Erstellung der Intercluster-Netzwerkschnittstellen können Sie Peering-Beziehungen mit anderen ONTAP Storage-Systemen herstellen.

. Wählen Sie *Add a data container*.
+
Sie können ein Volume zur Verwendung durch AIDE hinzufügen und es einem bestimmten SVM zuordnen.

. Wählen Sie *Add Workspaces*.
+
Dadurch wird die Data Engine-Arbeitsbereiche-Seite geöffnet, auf der Sie link:../workspaces/create-workspace.html["einen Arbeitsbereich erstellen"] können.

. Wählen Sie *Configure OpenID Connect*.
+
Sie müssen link:../install-setup/configure-oidc.html["OpenID-Authentifizierung konfigurieren"] aktivieren, um den Zugriff auf AIDE Console zu ermöglichen.



.Nachdem Sie fertig sind
Nachdem alle Aufgaben erledigt sind, wird das Hilfefenster ausgeblendet. Sie können es bei Bedarf manuell wieder öffnen und schließen.



== Aktivieren Sie Ihre AIDE Lizenz für NetApp DCN Bereitstellungen

Wenn Sie NetApp DCNs in Ihrer Konfiguration haben, müssen Sie link:../install-setup/install-license-task.html["Installieren Sie Ihre AIDE Premium-Servicelizenz"] nachdem Ihr AIDE-Cluster eingerichtet und mit dem AFX-Speichersystem integriert wurde. Für die vollständige AIDE-Funktionalität, einschließlich Vektorisierung und Guardrails-Funktionen, ist die AIDE Premium Services-Lizenz erforderlich.
